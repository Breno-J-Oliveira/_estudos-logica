## O que é esta categoria?

A categoria **Estudos de Lógica de Programação** documenta os primeiros passos na programação — o momento em que se aprende a **pensar como um programador**. São **7 projectos** que cobrem desde os primeiros algoritmos em Portugol/VisualG até ao domínio de Git e aos fundamentos da Programação Orientada a Objetos (POO).

É aqui que tudo começa: antes do JavaScript, antes do PHP, antes de qualquer framework. Aqui aprende-se a decompor problemas, a estruturar soluções com sequência/condição/repetição, e a organizar código com versionamento.

**A fundação sobre a qual todo o resto do repositório foi construído.**

---

## O que esta categoria entrega?

### Para o estudante de programação
- **Raciocínio algorítmico** — decompor problemas em passos lógicos
- **Estruturas de controlo** — sequência, condição (`se`/`senao`), repetição (`enquanto`, `para`)
- **Tipos de dados e variáveis** — inteiro, real, caractere, lógico
- **Operadores** — aritméticos, relacionais, lógicos
- **Entrada e saída** — `leia()` e `escreva()` em Portugol/VisualG
- **Versionamento com Git** — `init`, `add`, `commit`, `branch`, `merge`
- **Programação Orientada a Objetos** — classes, objetos, herança, encapsulamento, polimorfismo
- **Transposição de lógica** — mesmo algoritmo implementado em múltiplas linguagens (VisualG → JS → PHP)

### Para o recrutador / avaliador
- **Base sólida** — 7 projectos que provam domínio dos fundamentos da computação
- **Múltiplos paradigmas** — lógica estruturada (Portugol) + POO + versionamento
- **Aprendizagem verificável** — exercícios progressivos com complexidade crescente
- **Transição documentada** — do pseudocódigo (Portugol) para código real (JS, PHP)
- **Programa Avanze** — participação em programa externo de aceleração
- **Git desde o início** — boas práticas de versionamento aplicadas desde a primeira aula

## Como funciona? — Percurso de Aprendizagem

### 🔰 Fase 1 — Primeiros Algoritmos (Projectos 1–3)

```
1. Aula de Lógica 1 — 10 exercícios em VisualG
   Calculadora, média, conversão de temperatura,
   área do retângulo, antecessor/sucessor,
   dobro/metade, salário, conversão de moeda,
   aumento salarial, velocidade média
       ↓
2. Aula de Lógica 2 — Reforço e mais exercícios
   Problemas com complexidade ligeiramente maior
       ↓
3. Exercício Portugol (10/04/2025)
   Prática adicional de sintaxe e lógica
```

**Objectivo:** familiarizar-se com a sintaxe do Portugol/VisualG, dominar os conceitos de variáveis, entrada/saída e operações matemáticas básicas.

### 🟡 Fase 2 — Consolidação (Projectos 4–5)

```
4. Atividade Prática (03/04/2025)
   Aplicação dos conceitos aprendidos
       ↓
5. Atividade do Avanze
   Programa de aceleração — exercícios
   de lógica e introdução a conceitos
   computacionais
```

**Objectivo:** consolidar o raciocínio lógico com exercícios práticos e participar de um programa externo de aceleração (Avanze).

### 🟢 Fase 3 — Ferramentas e Paradigmas (Projectos 6–7)

```
6. Aula de Versionamento — Git
   init, add, commit, branch, checkout, merge
   Trabalho em equipa simulado
       ↓
7. Programação Orientada a Objetos (POO)
   Classes, objetos, herança, encapsulamento
   Implementações em JavaScript e PHP
```

**Objectivo:** aprender a gerir código com Git e migrar da lógica estruturada para o paradigma de orientação a objetos.

## Conceitos de POO Abordados

| Conceito | Descrição | Exemplo no projecto |
|----------|-----------|---------------------|
| **Classe** | Molde que define atributos e métodos | `class Carro`, `class Pessoa` |
| **Objeto** | Instância concreta de uma classe | `$meuCarro = new Carro()` |
| **Atributos** | Propriedades que descrevem o objeto | `$marca`, `$modelo`, `$ano` |
| **Métodos** | Comportamentos do objeto | `acelerar()`, `frear()` |
| **Encapsulamento** | Proteção de dados com `private`/`public` | `private $saldo` |
| **Herança** | Uma classe herda atributos/métodos de outra | `class Cachorro extends Animal` |
| **Polimorfismo** | Métodos com mesmo nome, comportamentos diferentes | `falar()` em `Cachorro` vs `Gato` |

## Como Usar Esta Categoria para Estudar

### Pré-requisitos
- **VisualG** (gratuito) para abrir ficheiros `.fprg`
- Navegador web para a atividade Avanze e os ficheiros JS
- **PHP** (opcional) para executar a atividade de POO em PHP
- **Git** (opcional) para replicar os comandos de versionamento
- Ferramenta de descompressão (WinRAR, 7-Zip) para ficheiros `.zip` e `.7z`

### Passo a passo

1. **Instala o VisualG** — é gratuito e necessário para executar os algoritmos
2. **Começa pelo Projecto 1** — extrai o `.zip` da `aulalogica`, abre cada `.fprg` no VisualG e executa (F9)
3. **Analisa cada exercício** — percebe o que cada linha faz, modifica valores, testa extremos
4. **Avança para a aulalogica2** — exercícios de reforço com complexidade ligeiramente maior
5. **Explora o versionamento** — lê o `GIT.txt` com as anotações e replica os comandos no terminal
6. **Estuda POO** — começa pelo `.js` (mais simples de executar no navegador) e depois compara com o `.php`
7. **Desafia-te** — tenta reescrever os exercícios de VisualG em JavaScript, como transição natural

### Estrutura de cada projecto

Cada subpasta contém:
- **`README.md`** (ou `readme.txt`) — objectivos, conceitos ensinados, explicação
- **Ficheiros de código** — `.fprg`, `.js`, `.php`, `.html`, `.css`
- **Arquivos compactados** — `.zip` ou `.7z` com os exercícios originais
